01

Programação Orientada a Objetos

Fundamentos e paradigmas da POO aplicados em C++

A Programação Orientada a Objetos é um paradigma que organiza o software em torno de objetos — entidades que combinam estado (atributos) e comportamento (métodos). Este módulo consolida os quatro pilares que sustentam todo o design orientado a objetos.

⬡

Encapsulamento

Oculta detalhes internos e expõe apenas o necessário por meio de interfaces bem definidas.

⬡

Herança

Permite que classes derivadas reutilizem e estendam comportamentos de classes base.

⬡

Polimorfismo

Objetos de tipos distintos respondem ao mesmo contrato de maneiras diferentes.

⬡

Abstração

Modela conceitos do mundo real destacando apenas características essenciais.

Tópicos abordados

  • Classes, objetos, construtores e destrutores
  • Modificadores de acesso: public, private, protected
  • Interfaces e classes abstratas
  • Sobrecarga e sobrescrita de métodos
  • Princípios SOLID como extensão da POO
02

Padrões de Projeto

Soluções reutilizáveis para problemas recorrentes de design

Documentados pelo Gang of Four, os padrões de projeto são receitas testadas para resolver problemas comuns de arquitetura. Eles promovem código mais legível, extensível e desacoplado.

Criacionais

  • Singleton
  • Factory Method
  • Abstract Factory
  • Builder
  • Prototype

Estruturais

  • Adapter
  • Bridge
  • Composite
  • Decorator
  • Facade

Comportamentais

  • Observer
  • Strategy
  • Command
  • Iterator
  • Template Method
03

Programação Multithread

Concorrência, paralelismo e sincronização de processos

Programas modernos precisam executar múltiplas tarefas simultaneamente. Este módulo apresenta os desafios da programação concorrente e as ferramentas para criar aplicações seguras, eficientes e livres de condições de corrida.

Conteúdo programático

  • Threads vs. Processos: diferenças e casos de uso
  • Criação e ciclo de vida de threads em Java (Thread, Runnable)
  • Sincronização: synchronized, Lock e Semaphore
  • Condições de corrida, deadlock e starvation
  • ExecutorService e Thread Pools
  • Programação assíncrona com CompletableFuture
⚡

Aviso: código concorrente mal escrito é uma das causas mais comuns de bugs difíceis de reproduzir. A disciplina dedica atenção especial a técnicas de depuração e testes.

04

Documentação UML

Modelagem visual de sistemas com a Unified Modeling Language

A UML é a linguagem padrão para documentar arquitetura de software. Através de diagramas bem elaborados, equipes comunicam decisões de design com precisão, reduzindo ambiguidades e acelerando o onboarding de novos membros.
